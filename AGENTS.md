## 3. 멀티 LLM 및 바이브 코딩 (Multi-LLM & Vibe Coding)

### 명시적 모델 라우팅 및 MCP 연동

Neo가 프롬프트 서두에 특정 태그를 입력한 경우, 설정된 VS Code MCP(Model Context Protocol) 서버 도구 또는 내부 라우팅 로직을 통해 최적의 모델/스크립트를 호출하여 작업을 수행합니다.
- 예: `@ga`, `@gv`, `@dual`, `@llama`, `@hermes`, `@qc`, `@qk`, `@ga_n`, `@ga_d`, `@ga_r`, `@gr_f`, `@v3`, `@r1`, `@v1`, `@n1`, `@gem` 등
- *지침*: `@ga` 호출 시 워크스페이스 내 `groq_mcp.py`의 `ga` 함수 구동 도구를 사용하고, `@gv`는 `gv` 함수를 실행하여 처리합니다.

### 지능적 모델 라우팅 및 자율 최적화 (Smart Routing)

1.  **작업 특화 최적화 (Task-Specific Selection)**:
    - 언어 리팩토링, UI 설계, 보안 검수 등 전담 전문가가 필요한 경우, 명시된 인프라 내에서 가장 적합한 전용 모델을 자율적으로 선택(또는 제안)하여 품질을 극대화합니다.
2.  **자동 Failover 및 복원력 (Auto Failover)**:
    - 특정 API/도구 호출 실패 시 사전에 정의된 Failover 시퀀스(**@ga ➔ @gv ➔ @dual ➔ @r1 ➔ @hermes**)에 따라 가용한 대체 수단을 찾아 우회 실행합니다.
3.  **투명한 실행 보고 (Transparent Reporting)**:
    - 모델/도구가 변경되거나 Failover가 발생한 경우, 최종 리포트에 **[수정/Failover 사유]**와 **[실제 가동 모델/도구]**를 명시합니다.
4.  **RAG 지식 검색 부재 시 Fallback 원칙 (중요)**:
    - **Neo4j** 지식베이스 등에 관련 데이터가 없을 경우, 에이전트 내장 지식(웹 검색 도구 사용 가능 시 포함)을 총동원하여 답변을 구성하고, 본문 하단에 `출처: [출처명](URL)` 형식으로 실제 지식 출처를 명확히 표시합니다.

### 바이브 코딩(Vibe Coding) 최적화 프로세스

1. **Vibe Translation**: Neo의 모호한 요청을 기술적 요구사항과 제품 사양으로 즉시 변환합니다.
2. **Visual Blueprint**: 구현 전, UI 컨셉과 아키텍처 구조를 VS Code Markdown 뷰어에서 확인하기 좋게 먼저 제시합니다.
3. **Modular Implementation**: VS Code 파일 쓰기 권한을 활용하여, PL이 생략 없이 즉각적으로 파일을 생성하고 전체 코드를 작성합니다.

## 5. 핵심 코딩 및 시스템 제어 원칙 (Core Rules)

- **완전한 코드 작성 (NO TRUNCATION)**: 어떠한 경우에도 생략(`...` 또는 `// 나머지 코드 동일`) 없이 **전체 소스코드(Full Code)**를 작성하여 파일에 적용합니다. 
- **언어별 표준 환경 준수 (터미널 제어 포함)**:
  - **Python**: **모든 패키지 설치 및 가상환경 관리는 오직 `uv` 명령어(`uv init`, `uv add`, `uv run` 등)만을 사용**합니다. (pip, conda 사용 지양)
  - **Web**: **Tailwind CSS, Shadcn/ui** 유틸리티 클래스를 최우선 활용합니다.
- **보안 및 접근성**: SQL 인젝션 방어, Parameterized Query, 보안 주석 필수.
