# HNIX Mate App 프롬프트 및 요구사항 정리

이 문서는 프로젝트 개발 시 사용된 주요 요청 사항들을 정리한 목록입니다. 추후 다른 AI 모델을 통해 고도화 작업을 수행할 때 참고하십시오.

## 1. 프로젝트 초기 구성 (기본 요구사항)
- **목표**: 회사 내 챗봇과 연동하여 직원들에게 복리후생과 회사 규칙을 조회 및 공유할 수 있는 웹 사이트 제공.
- **기술 스택**: Vue3 (Composition API), Vite, Bootstrap 5, Bootstrap Icons, Axios, ESLint (Standard).
- **디자인 가이드**:
  - 기업용 대시보드 스타일의 깔끔한 디자인.
  - 회사 CI 색상 조화롭게 사용: `rgb(25, 58, 148)` (Primary), `rgb(19, 156, 69)` (Secondary).
  - 반응형 웹 디자인 적용.
- **주요 화면 구성**:
  - **메인 화면**: "무엇이든 물어보세요" 문구와 넓은 검색바. 하단에 주요 카테고리(휴가/연차, 의료비 등) 아이콘 카드 배치. 검색 시 대화 내용과 관련 규정(PDF/링크) 우측 표시.
  - **파일 올리기 화면**: 챗봇 학습을 위한 파일 업로드 기능.

## 2. 화면 이동 및 전역 스타일
- **애니메이션**: 페이지 이동(View 간 전환) 시 부드러운 전환 효과(애니메이션) 추가.

## 3. 에러 핸들링 및 모달 시스템
- **요구사항**: API 연동 과정에서 발생하는 서버 오류나 네트워크 장애 시, 단순히 `alert`를 띄우는 대신 사용자에게 친절한 안내와 재시도 기회를 제공하는 전역 에러 모달 창 구현.
- **세부사항**: `useErrorModal` 컴포저블을 통한 상태 관리 및 Bootstrap 기반의 커스텀 모달 컴포넌트(`ErrorModal.vue`) 제작.

## 4. UI 고도화 및 테마 적용
- **동적 배경**: IT 어시스턴트 느낌을 주기 위해 메인 화면에 떠다니는 아이콘과 부드러운 그라디언트가 포함된 동적 배경 애니메이션 추가.
- **디자인 스타일**: 카드와 입력창에 **글래스모피즘(Glassmorphism)** 스타일(투명 배경 + 블러 효과)을 적용하여 배경과 조화를 이루면서도 고급스러운 분위기 조성.
- **상단바 제어**: 메인 화면 진입 시에는 상단바를 숨기고, 실제 채팅이 시작되거나 다른 화면으로 이동할 때만 상단바를 표시하도록 레이아웃 조정.

## 5. 이메일 전송 기능 고도화
- **요구사항**: AI의 답변 내용을 직원들이 동료에게 공유할 수 있도록 이메일로 전송하는 기능 구현.
- **기능 상세**:
  - 다중 수신자를 관리할 수 있는 이메일 전송 모달(`EmailModal.vue`) 구현.
  - 사용자가 수신자 메일을 추가/삭제할 수 있는 인터페이스 제공.
  - 전송 시 정해진 포맷(`받는 사람: {list}, 제목: Send Mail by HNIX AI assistant, 내용: {AI답변}`)으로 Webhook API 호출.

## 6. 파일 업로드 및 유효성 검사
- **요구사항**: 챗봇 학습용 파일을 서버에 업로드하는 기능 실체화.
- **보안 및 정책**:
  - 파일당 최대 10MB 용량 제한.
  - 허용 확장자 제한: PDF, Word (.doc, .docx), PowerPoint (.ppt, .pptx)만 업로드 가능하도록 유효성 검사 추가.
  - 유용한 피드백: 유효하지 않은 파일 시도 시 사유(용량 초과 또는 미지원 형식)를 정확하게 안내.

## 7. 파일 정리 및 구조 개편
- **요청 내용**: 불필요한 개발 히스토리 파일 삭제, 프롬프트 이력 파일 이름 변경 및 컴포넌트 구조 체계화.
- **수행 작업**:
  - `DEVELOPMENT_HISTORY.md` 삭제.
  - `prompt-list.md` -> `prompt-history.md`로 변경 및 최신 작업 내역 기록 유지.
  - `src/components` 내부를 `common/`, `home/` 하위 폴더로 분류하여 정리.
  - 모든 소스 코드의 Import 경로를 새로운 구조에 맞춰 일관성 있게 업데이트.

## 8. 카테고리 연동 및 UI 개편 (그린 테마)
- **요청 내용**: 카테고리 8종 웹훅 연동, 메인 화면 버튼 개편(5개), 채팅방 UI 그린 테마 적용 및 동적 배경 추가, CI 로고 노출.
- **수행 작업**:
  - **웹훅 연동**: `chatbotWebhook.js`에서 `class` 파라미터를 통해 8종 카테고리(General, Ceo_message 등)를 서버에 전달하도록 로직 강화.
  - **메인 화면 개편**: 자료실, 사내소식, 상조, 사내규정, 영수증 5개 버튼으로 재배치하고 관련 아이콘 최적화 및 CI 로고 배치.
  - **채팅방 고도화**:
    - 상단에 실시간 카테고리 선택 및 상태 표시 바 추가.
    - 전체 색상을 HNIX CI 녹색(`rgb(19, 156, 69)`)으로 변경 및 글래스모피즘 스타일 적용.
    - 채팅방 배경에 유동적인 Blob 애니메이션을 적용하고, 카테고리에 따라 배경색 테마가 변하도록 구현.
    - 채팅창 곳곳에 HNIX 로고 및 브랜딩 요소 추가.
  - **전역 스타일**: `base.css`를 수정하여 스크롤바와 주요 강조색을 초록색 테마로 일관되게 적용.

## 9. UI 피드백 반영 및 디테일 개선
- **요청 내용**: 전면적인 CI 로고 이미지 교체, 채팅방/메인 배경 애니메이션 수정(복원 및 가시성 해결), 전송 버튼 스타일 일치.
- **수행 작업**:
  - `public/HNIX-CI.png` 이미지를 `index.html` 파비콘(탭 메뉴 아이콘), `HomeView.vue` 헤더와 중앙부, `ChatWindow.vue` 하단 푸터 로고에 일괄 적용.
  - `HomeView.vue`에서 채팅방 렌더링 시 배경색 간섭을 없애기 위해 `.chat-screen` 영역 배경 투명화 처리. 결과적으로 채팅창의 Blob 애니메이션이 정상 표시됨.
  - `HomeView.vue`에 누락되었던 둥둥 떠다니는 아이콘(`.icon-1` ~ `.icon-10`)들의 CSS 위치/가시성 로직을 복원하여 웰컴 화면 생동감 되살림.
  - `SearchBar.vue`의 전송 버튼 영역을 초록색 테마(그라데이션 및 hover 효과)로 리디자인하고, 직관적인 방향표시 아이콘 대신 전송 아이콘(`bi-send-fill`)으로 교체.

## 10. 2차 UI 피드백 반영 (메인 배경 및 채팅방 고도화)
- **요청 내용**: 메인 배경 움직이는 아이콘 증가(20개) 및 파란/초록색 배경 전환, 채팅 단 로고 크기 축소, 카테고리별 채팅방 배경색 가시성 확보, 대화 시 스크롤 문제 해결.
- **수행 작업**:
  - `HomeView.vue`의 배경 그라데이션(`.animated-background`)을 파란색(CI 보조색) ↔ 초록색(CI 주요색)으로 번갈아 전환하도록 CSS 애니메이션 변경.
  - 메인 화면에 둥둥 떠다니는 아이콘을 기존 10개에서 20개로 추가하고 파란색/초록색 계열 색상을 조화롭게 배치.
  - `ChatWindow.vue` 하단 푸터의 CI 로고 높이를 `14px`로 줄여 헤더 텍스트와 위화감이 없도록 조절.
  - `HomeView.vue`의 `.home-container` 배경색을 투명으로 변경하고 채팅창 배경 틴트의 채도를 높여, 카테고리(General, Resource 등) 변경 시마다 배경색상이 확연히 변하도록 적용.
  - `ChatWindow.vue` 전체 컨테이너에 `flex: 1 1 auto; min-height: 0;` CSS를 추가하여 AI와 장문 대화 시 `.messages-area` 내부에서 정상적으로 스크롤바가 생성 및 작동되도록 해결.

## 11. 3차 UI 피드백 세부 개선
- **요청 내용**: 브라우저 탭 아이콘을 챗봇 모양으로 변경, 메인화면 영수증 버튼 제거 밑 색상 변경(투명도 감소, 농도 짙게), 채팅방 헤더 로고 겹침 해결 및 배경 통일 공유, 카테고리 노출 주석처리 및 로딩 애니메이션 복구.
- **수행 작업**:
  - `public/bot-favicon.svg`를 로봇 심볼 SVG로 만들어 `index.html` 파비콘에 적용.
  - 메인 화면의 카테고리 배열(`mainCategories`)에서 '영수증'을 제거하여 4개 아이템으로 맞추고, `CategoryCard.vue`의 배경 투명도를 `0.5`에서 `0.85`로 변경(`opacity` 증가).
  - 전체 CSS의 그라데이션 컬러 파레트(`#bbdefb` 활용)를 더 진한 파란-녹색 톤으로 상승 시킴.
  - `HomeView.vue` 헤더의 로고 박스 하드 코딩 크기 제한(`width:40px`)을 풀어 텍스트 겹침 증상을 해소. 채팅 헤더와 푸터에 화이트 대비 시야를 덜 뺏아가게 연녹색 블러 배경 지정.
  - 기존 웰컴 화면에 종속되던 `animated-background` 배경 요소 20개를 뷰의 최상단 공통 컨테이너로 이동하여, 채팅 뷰와 메인 뷰가 완벽히 같은 배경 애니메이션을 표시하도록 구성.
  - 채팅창 상단의 서브 카테고리 선택 바 HTML 코드(`category-sticky-bar`) 숨김/주석 유지 반영.
  - 소실된 타이핑(`.typing-indicator .dot` 과 통통 튀는 keyframe) CSS를 `ChatWindow.vue`에 다시 삽입하여 응답 로딩 시 애니메이션 복원.

## 12. 4차 UI 피드백 세부 개선
- **요청 내용**: 메인화면/업로드뷰/채팅 테마 레이아웃 분리(메인 파스텔톤, 채팅/업로드는 진록 고정), 메인 폰트 가독성 상향, 카테고리 투명도 상향, 채팅방 스크롤 재설정, "Powered by HNIX AI" 위치 이동, 트리거 메시지 숨김 처리, 업로드 화면 뷰 동기화 및 카테고리 셀렉터 추가.
- **수행 작업**:
  - `HomeView.vue`의 배경 `.animated-background` 컬러 코드를 연한 파스텔 민트/블루 톤으로 변경하고, 메인 제목과 설명 글귀에 `text-dark-slate`, `text-slate-muted`와 같은 진회색 계열 폰트 컬러를 설정함. `CategoryCard.vue` 버튼의 배경 투명도를 조금 더 맑게(`0.6`) 수정함.
  - 채팅방(`messages.length > 0`) 진입 시 동적 그라데이션 대신 단일 짙은 녹색(`#1b4d2a`) 배경이 나타나도록 `:class="{'chat-mode-bg'}"` 상태 분기를 걸고, `flex h-100` 및 부모 요소에 `max-height: 100vh; overflow:hidden` 보강으로 스크롤이 창 밖으로 밀리지 않도록 고정.
  - `ChatWindow.vue` 하단의 HNIX AI 로고 컨테이너를 없애고 대신 `HomeView.vue`의 `<SearchBar>` 컨테이너 바로 아래로 옮겨 화면 레이아웃 하단에 위치시킴.
## 13. 6차 문제해결 (핫픽스)
- **요청 내용**: 메인화면에서 카드(카테고리) 선택 시, 채팅화면으로 이동은 하되 AI에게 자동으로 메시지를 보내지 않도록 수정.
- **수행 작업**:
  - `HomeView.vue`에서 기존 화면 전환 기준이었던 `messages.length > 0` 조건을 `isChatActive`라는 별도의 Boolean 상태 변수로 분리.
  - 메인 카테고리 카드 클릭 이벤트 핸들러 `handleCategorySelect` 내에서, 챗봇 Webhook으로 초기 질문 문구("~ 관련 정보를 알려주세요.")를 전송하던 기존 `handleSearch` 호출 코드를 삭제(메시지 배열 추가 방지).
  - 단순히 화면만 `isChatActive = true`로 전환하여, 텅 빈 채팅창(질문 유도 안내문이 있는 빈 채팅룸)에만 진입하도록 수정함.
