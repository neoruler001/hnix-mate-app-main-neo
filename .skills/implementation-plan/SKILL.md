# 📋 Implementation Plan & Progress Tracking Guide

이 문서는 새로운 기능 개발이나 버그 수정을 시작하기 전, **구현 계획(Implementation Plan)을 세우고 진행 상황을 추적하기 위한 표준 가이드**입니다. 

주니어 개발자가 작업의 큰 그림을 이해하고, 막히는 부분을 빠르게 파악하며, 팀원들과 투명하게 진행 상황을 공유하는 것을 목적으로 합니다.

---

## 🎯 왜 구현 계획을 작성해야 하나요? (주니어 개발자를 위한 팁)
1. **길 잃음 방지:** 코딩부터 시작하면 세부 구현에 빠져 목적을 잊기 쉽습니다. 계획을 먼저 세우면 내비게이션 역할을 해줍니다.
2. **작업 쪼개기 (Divide & Conquer):** 막막해 보이는 큰 작업도 UI, 상태 관리, API 연동 등 작은 단위로 쪼개면 해볼 만한 크기가 됩니다.
3. **진척도 시각화:** 내가 하루 종일 무엇을 했는지, 현재 어디에 멈춰있는지 팀원(특히 사수)에게 쉽게 공유할 수 있습니다.
4. **이슈 사전 차단:** 계획을 세우는 단계에서 '어? 이 데이터는 어디서 가져오지?' 같은 막히는 부분을 미리 발견할 수 있습니다.

---

## 🚦 진행 상태(Status) 표기 표준

작업의 진행 상황은 체크박스와 상태 태그를 결합하여 한눈에 파악할 수 있게 업데이트합니다. 매일 업무 시작 전후로 업데이트하는 것을 권장합니다.

* `[ ]` ⚪ **TODO**: 아직 시작하지 않음
* `[-]` 🟡 **IN PROGRESS**: 현재 작업 중
* `[x]` 🟢 **DONE**: 작업 완료 및 자체 테스트 통과
* `[!]` 🔴 **BLOCKED**: 외부 요인(API 미완성, 기획 변경, 원인 모를 에러 등)으로 인해 작업이 막힘
