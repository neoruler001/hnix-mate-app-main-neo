# 테스트 및 검증 Skill

## 1. 린팅 및 포맷팅 (ESLint/Stylelint)
- 프로젝트에 설정된 ESLint 규칙에 맞춰 코드를 작성합니다.
- 빌드 전 자동 린트 과정을 통해 잠재적 오류를 사전에 차단합니다.

## 2. 테스트 구조 및 패턴
- **AAA 패턴**: 테스트 코드를 Arrange(준비), Act(실행), Assert(검증)의 세 단계로 구조화합니다.
- **독립성 및 멱등성**: 각 테스트는 독립적으로 실행 가능해야 하며, 반복 실행 시 결과가 동일해야 합니다.
- **Mocking**: API 호출 등 외부 의존성은 반드시 머킹 처리하여 테스트 속도와 안정성을 확보합니다.

## 3. 검증 전략
- **커버리지**: 단순한 라인 커버리지보다 비즈니스 로직의 분기점(Branch) 커버리지를 높이는 데 집중합니다.
- **유효성 검사**: 프론트엔드에서는 폼 입력값 및 데이터 형식에 대한 1차 검증을 철저히 수행합니다.
